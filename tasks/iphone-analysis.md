### Key observations from code review

1. **CORS is already wide open** (`main.py:21-26`):
   ```python
   app.add_middleware(
       CORSMiddleware,
       allow_origins=["*"],
       allow_methods=["*"],
       allow_headers=["*"],
   )
   ```
   A Capacitor app calling the API from a native shell will succeed without any changes.

2. **Jinja2 usage is minimal** ‚Äî only one server-side variable:
   ```html
   <!-- index.html:20 -->
   <h1 id="child-name-title">ü¶ï {{ config.child_name }} Learns!</h1>
   ```
   JavaScript immediately overwrites this from `localStorage` on page load, so this is cosmetic only.

3. **All API calls are relative** (`/api/word`, `/api/tts`, etc.) ‚Äî these resolve correctly when the web content is loaded from the Cloud Run URL in a remote-server Capacitor configuration.

4. **MediaRecorder has iOS fallback logic already**:
   ```javascript
   // app.js ‚Äî already handles codec detection
   const options = MediaRecorder.isTypeSupported('audio/webm;codecs=opus')
     ? { mimeType: 'audio/webm;codecs=opus' }
     : {}
   mediaRecorder = new MediaRecorder(stream, options)
   ```
   On iOS, this falls back to `audio/mp4` which is supported in WKWebView (Capacitor's iOS webview) from iOS 14.3+.

5. **Web Audio API SFX** ‚Äî all synthesized, no audio files. iOS requires a user-gesture before AudioContext can start, but all SFX is already triggered by button taps (user gestures), so this works as-is.

6. **TTS audio playback** ‚Äî the app plays `<Audio>` objects created from API URL streams. iOS requires user-gesture before autoplay, but all TTS is triggered by user actions (play button, record button). This is compliant.

## 3. Approach Options

### Option A: Capacitor (Remote Server Mode) ‚Äî RECOMMENDED

**What it is:** Capacitor wraps the existing web app in a native iOS WKWebView shell. In remote-server mode, the WKWebView loads the app directly from the Cloud Run URL ‚Äî exactly like a browser, but without browser chrome and with a native app icon.

**Changes required:**
- Create `package.json` (1 new file, ~15 lines)
- Create `capacitor.config.ts` (1 new file, ~15 lines)
- On a Mac: `npx cap add ios` generates the iOS project
- Add microphone permission string to `ios/App/App/Info.plist`
- Add app icons + launch screen in Xcode

**Pros:**
- Backend: zero changes
- Frontend: zero changes
- Familiar web development workflow
- Fast iteration (change backend ‚Üí see it in app immediately)
- Native microphone permission prompts handled automatically

**Cons:**
- Requires internet connection (app loads from server)
- Apple occasionally rejects "web wrapper" apps ‚Äî mitigated by having genuine native hardware use (microphone)
- Requires macOS + Xcode for final build + submission

---

### Option B: PWA (Progressive Web App)

**What it is:** Add a web app manifest and service worker so iPhone users can "Add to Home Screen" via Safari, getting an app-like experience.

**Changes required:**
- `static/manifest.json` (new, ~25 lines)
- `static/js/sw.js` service worker (new, ~40 lines)
- 3 meta tags in `index.html`
- Route to serve `/manifest.json` in `main.py`

**Pros:**
- Zero native tooling required
- Works immediately

**Cons:**
- Cannot be published to the App Store
- Safari PWA limitations: no background processing, limited push notifications
- MediaRecorder on iOS Safari (outside WKWebView) has more restrictions than in Capacitor
- Users must know to "Add to Home Screen" ‚Äî less discoverable

---

### Option C: React Native / Swift Native

**What it is:** Full frontend rewrite in Swift (SwiftUI) or React Native, calling the same backend API.

**Changes required:** Complete frontend rewrite (~1000+ lines new code)

**Pros:**
- Best native performance and feel
- Full access to all native iOS APIs
- No App Store rejection risk for web wrapper

**Cons:**
- Not "minimum changes" ‚Äî this is a full rewrite
- Requires Swift/React Native expertise
- Backend stays the same but frontend is entirely new

## 4. Recommended Implementation: Capacitor Remote Server

### Prerequisites (one-time setup)
- macOS computer (required for Xcode)
- Xcode installed (free from Mac App Store) ‚Äî requires macOS 14+
- Node.js 18+ installed (`brew install node`)
- Apple Developer Account ($99/year at developer.apple.com)
- The app deployed to Cloud Run at a stable URL

### Step 1: Create `package.json`

Create this file at the project root:

```json
{
  "name": "myra-language-teacher",
  "version": "1.0.0",
  "description": "Toddler language learning app for Myra",
  "scripts": {
    "cap:sync": "npx cap sync",
    "cap:ios": "npx cap open ios",
    "cap:build": "npx cap sync ios"
  },
  "dependencies": {
    "@capacitor/core": "^6.0.0",
    "@capacitor/ios": "^6.0.0"
  },
  "devDependencies": {
    "@capacitor/cli": "^6.0.0"
  }
}
```

### Step 2: Create `capacitor.config.ts`

```typescript
import { CapacitorConfig } from '@capacitor/cli';

const config: CapacitorConfig = {
  // Must match the Bundle ID you register in App Store Connect
  appId: 'com.myra.languageteacher',
  appName: 'Myra Learns!',

  // This tells Capacitor to load from the remote server,
  // not from local static files. ALL existing relative API
  // calls (/api/word, /api/tts, etc.) continue to work.
  server: {
    url: 'https://YOUR-CLOUD-RUN-URL.run.app',
    cleartext: false,
  },

  // iOS-specific settings
  ios: {
    // Allows the WKWebView to access the microphone
    // (the actual permission string goes in Info.plist)
    backgroundColor: '#FFE4F0',
  },
};

export default config;
```

**Replace `YOUR-CLOUD-RUN-URL`** with the actual Cloud Run service URL from:
```bash
cd infra && terraform output service_url
# or
gcloud run services describe dino-app --region=us-central1 --format='value(status.url)'
```

### Step 3: Install Capacitor and initialize iOS project (on Mac)

Run these commands at the project root:

```bash
# Install npm dependencies
npm install

# Initialize Capacitor with the iOS platform
npx cap add ios

# Sync (copies config to iOS project)
npx cap sync
```

This generates an `ios/` directory containing the full Xcode project.

### Step 4: Add microphone permission to `Info.plist`

Open `ios/App/App/Info.plist` and add:

```xml
<key>NSMicrophoneUsageDescription</key>
<string>Myra Learns uses the microphone so you can practice saying words out loud and check your pronunciation.</string>
```

This message is shown to the user when iOS prompts for microphone access.

### Step 5: Open in Xcode and configure

```bash
npx cap open ios
```

3. **Set version**: General ‚Üí Version: `1.0.0`, Build: `1`
4. **Set deployment target**: iOS 14.3 minimum (required for MediaRecorder support)

## 7. Potential Issues and Mitigations

### Issue 1: Apple "web wrapper" rejection
**Risk:** Apple may reject the app if reviewers decide it's "simply a wrapper around a website."

**Mitigation:**
- In Review Notes, emphasize: *"This app uses native iOS microphone access for real-time speech recognition to check children's pronunciation. The core functionality (record audio ‚Üí send to server ‚Üí get feedback ‚Üí animate dino) cannot be achieved with a simple browser bookmark."*
- The microphone use distinguishes this from a pure web wrapper
- Many successful App Store apps use WKWebView with a backend

### Issue 2: MediaRecorder audio format on iOS
**Risk:** `audio/webm;codecs=opus` is not supported in WKWebView on iOS.

**Status:** Already handled. The app's codec detection falls back to the system default (`audio/mp4`) when WebM is unavailable. The backend's pydub/ffmpeg pipeline handles `audio/mp4` input.

**Verify:** After installing on iPhone, test the "Say It!" feature and check that the backend logs show `audio/mp4` being received and processed successfully.

### Issue 3: App requires internet (no offline mode)
**Risk:** No internet = no app functionality.

**Impact:** Low ‚Äî this is a teaching app for home use where WiFi is available.

**If needed later:** Add a service worker + local caching of word data and pre-generated TTS audio. This is not required for initial launch.

### Issue 4: Cold start delay (Cloud Run scales to zero)
**Risk:** If the Cloud Run instance has scaled to zero, the first user action after a long idle period may take 10‚Äì30 seconds.

**Status:** The Whisper model is pre-loaded in the Docker image (baked in during build), so this is primarily network cold start, not model loading.

**Mitigation options:**
- Set `min_instance_count = 1` in `infra/cloud_run.tf` (increases cost ~$20/month)
- Show a loading spinner while waiting for the server to respond
- Accept the cold start (family use case ‚Äî happens rarely)

### Issue 5: gTTS requires internet on the server
**Risk:** If Google's TTS service is unreachable, "Hear It!" and voice prompts fail silently.

**Status:** Not an iOS-specific issue ‚Äî same risk on web.

## 8. PWA Alternative (No App Store)

If you want Myra to use the app on her iPhone **without going through the App Store**, a PWA ("Add to Home Screen") is faster to set up.

### What to add

**1. Serve a web manifest** ‚Äî add to `main.py`:
```python
from fastapi.responses import FileResponse

@app.get("/manifest.json")
async def manifest():
    return FileResponse("static/manifest.json", media_type="application/manifest+json")
```

**2. Create `static/manifest.json`:**
```json
{
  "name": "Myra Learns!",
  "short_name": "Myra Learns",
  "description": "Learn Telugu and Assamese with Roo the dino",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#FFE4F0",
  "theme_color": "#FF69B4",
  "orientation": "portrait",
  "icons": [
    {
      "src": "/static/icons/icon-192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "/static/icons/icon-512.png",
      "sizes": "512x512",
      "type": "image/png"
    },
    {
      "src": "/static/icons/icon-1024.png",
      "sizes": "1024x1024",
      "type": "image/png",
      "purpose": "any maskable"
    }
  ]
}
```

**3. Add meta tags to `templates/index.html`** (inside `<head>`):
```html
<!-- PWA / iOS home screen support -->
<link rel="manifest" href="/manifest.json" />
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-status-bar-style" content="default" />
<meta name="apple-mobile-web-app-title" content="Myra Learns!" />
<link rel="apple-touch-icon" href="/static/icons/icon-192.png" />
```

**4. Create icons** (192√ó192, 512√ó512, 1024√ó1024 PNG) of the dino and place in `static/icons/`.

### How to install on iPhone
1. Open Safari on iPhone
2. Navigate to `https://your-cloud-run-url.run.app`
3. Tap the Share button (box with arrow pointing up)
4. Tap "Add to Home Screen"
5. Tap "Add"

The app now appears on the home screen with a full-screen experience (no Safari address bar), working exactly like an app.

**Limitation:** PWAs are not discoverable via the App Store. This is fine for personal/family use but not for public distribution.
